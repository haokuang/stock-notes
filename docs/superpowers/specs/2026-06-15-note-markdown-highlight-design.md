# 笔记 Markdown 阅读与永久高亮设计

## 目标

在笔记详情页提供接近微信读书的阅读体验：

- 普通笔记和 MD 文档都按 Markdown 正确排版，不再直接显示 `##`、`**` 等源码。
- H5 支持鼠标拖选或移动端长按选中文字后显示浮动工具条。
- 工具条首版只提供“高亮”和“复制”。
- 高亮使用单一黄色，永久保存。
- 点击已有高亮可取消高亮或复制文字。
- 文档编辑后，仍能匹配原文的高亮自动保留，无法可靠匹配的高亮自动删除。
- 微信小程序正确展示 Markdown 与已有高亮，并使用系统原生文字选择和复制。
- 本版不考虑抖音小程序的交互适配。

## 范围边界

高亮仅作用于详情页正文，不包含标题、股票标签、时间、AI 总结、图片和价格卡片。

H5 提供完整的新增、取消和复制体验。微信小程序没有可靠的原生选区范围回调，因此不实现自定义高亮工具条，也不允许在微信端创建或取消高亮；微信端使用 `RichText` 的 `userSelect` / `selectable` 提供系统复制，并展示用户已在 H5 创建的高亮。

首版不支持：

- 多种高亮颜色。
- 高亮批注、想法或标签。
- 跨段重叠高亮。
- 高亮列表、全文导航。
- 多人共享高亮。
- 标题和图片高亮。

## 总体架构

正文渲染和高亮定位以“渲染后纯文本”为统一坐标系。

后端负责：

1. 将笔记正文转换成经过清洗的 HTML。
2. 从 HTML 提取规范化纯文本并计算 SHA-256 内容指纹。
3. 根据文本引用锚点重定位已有高亮。
4. 删除无法唯一匹配的失效高亮。
5. 将有效高亮注入 HTML，返回可直接展示的 `rendered_content`。

H5 负责：

1. 将 `rendered_content` 注入正文容器。
2. 监听浏览器 Selection API，读取选中文字、字符位置和工具条坐标。
3. 调用接口创建高亮或复制文本。
4. 通过事件代理识别已有高亮，支持取消和复制。

微信小程序负责：

1. 使用 `RichText` 展示 `rendered_content`。
2. 开启原生文字选择与复制。
3. 不自行计算选区和高亮位置。

## 数据模型

新增 `note_highlights` 表：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | varchar(36) | 主键 |
| `user_id` | uuid | 用户隔离，外键指向 `auth.users` |
| `note_id` | varchar(36) | 所属笔记，删除笔记时级联删除 |
| `selected_text` | text | 被高亮的原文 |
| `prefix_text` | text | 选区前最多 32 个字符 |
| `suffix_text` | text | 选区后最多 32 个字符 |
| `start_offset` | integer | 在渲染后纯文本中的起点 |
| `end_offset` | integer | 在渲染后纯文本中的终点 |
| `source_hash` | varchar(64) | 创建或最后重定位时的正文指纹 |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |

约束：

- `selected_text` 非空。
- `start_offset >= 0`。
- `end_offset > start_offset`。
- 同一笔记、同一正文版本下不允许重复或重叠高亮。
- 表开启 RLS，用户只能操作自己的高亮。

## 高亮锚点与重定位

创建高亮时保存三类信息：

1. Text Position Selector：`start_offset` / `end_offset`。
2. Text Quote Selector：`selected_text`。
3. Context Selector：`prefix_text` / `suffix_text`。

页面重新打开时：

1. 如果 `source_hash` 与当前正文一致，且原偏移处仍等于 `selected_text`，直接使用原位置。
2. 否则在当前正文中搜索全部相同 `selected_text`。
3. 根据前后文匹配程度评分，距离旧位置更近者作为次级排序条件。
4. 只有唯一最高分候选才保留，并更新偏移和指纹。
5. 没有候选或最高分并列时判定为不可靠，删除该高亮。

该策略允许正文前面插入或删除内容后继续保留原高亮，同时避免把重复句子错误标记到其他位置。

## Markdown 渲染

数据库中的原始编辑内容保持不变：

- `type=doc` 优先使用 `doc_md`。
- 普通笔记使用 `content` 作为 Markdown 源。

后端使用现有 `marked` 转换 Markdown，并继续使用 `DOMPurify` 清洗。自动简评等历史 HTML 内容允许由 Markdown 渲染器透传后再次清洗。

后端新增 HTML 文本树处理能力，用于：

- 提取与浏览器 `textContent` 一致的纯文本。
- 按字符区间拆分文本节点。
- 注入 `<span data-highlight-id="...">` 高亮节点。

高亮使用内联背景色，确保微信 `RichText` 不依赖 H5 CSS 也能显示。

## API

### 获取笔记详情

`GET /api/notes/:id`

在现有数据上新增：

```json
{
  "rendered_content": "<p>...</p>",
  "content_hash": "sha256",
  "highlights": [
    {
      "id": "uuid",
      "selected_text": "被高亮文字",
      "start_offset": 10,
      "end_offset": 15
    }
  ]
}
```

读取详情时完成高亮重定位和失效清理。

### 创建高亮

`POST /api/notes/:noteId/highlights`

```json
{
  "selected_text": "被高亮文字",
  "prefix_text": "前文",
  "suffix_text": "后文",
  "start_offset": 10,
  "end_offset": 15,
  "source_hash": "sha256"
}
```

后端重新生成当前正文纯文本并校验：

- 内容指纹一致。
- 偏移对应文字一致。
- 不与当前有效高亮重叠。

正文已变化时返回 HTTP 409，前端重新加载详情并提示用户重新选择。

### 删除高亮

`DELETE /api/notes/:noteId/highlights/:highlightId`

仅允许删除当前用户、当前笔记下的高亮。

## H5 交互

选择完成后，浮动工具条显示在选区上方；空间不足时显示在下方，并限制在视口内。

工具条包含：

- 高亮：创建黄色高亮，成功后重新加载或局部更新正文。
- 复制：使用 `Taro.setClipboardData`，成功后关闭工具条。

点击已有黄色高亮时显示：

- 取消高亮：删除记录并移除页面标记。
- 复制：复制该高亮文字。

以下情况不显示工具条：

- 空选区或仅空白。
- 选区不完全位于正文容器。
- 选区跨越已有高亮或与已有高亮重叠。
- 选区来自标题、卡片或其他页面区域。

滚动、点击正文外部、页面隐藏或选区清空时关闭工具条。

## 微信小程序交互

使用 Taro `RichText`：

- `nodes={rendered_content}`
- `userSelect`
- `selectable`

微信端展示标题、段落、列表、引用、代码、表格及黄色高亮。长按文字使用微信系统选择菜单复制，不展示自定义工具条。

## 错误处理

- 创建高亮返回 409：提示“正文已更新，请重新选择”，并重新加载。
- 创建或删除失败：保留当前页面内容并显示 toast。
- Markdown 渲染失败：记录服务端错误，详情接口退回纯文本安全 HTML。
- 高亮定位失败：删除失效记录，不影响正文加载。
- 单条异常高亮不能阻断其他高亮渲染。

## 测试策略

后端纯函数测试：

- Markdown 转安全 HTML。
- HTML 纯文本提取。
- 原偏移直接命中。
- 文本前移或后移后的重定位。
- 重复文本通过前后文唯一定位。
- 歧义文本判定失效。
- 高亮 HTML 注入不破坏标签结构。

后端集成测试：

- 用户只能读取和修改自己的高亮。
- 创建、删除、笔记级联删除。
- 重叠高亮被拒绝。
- 旧内容指纹返回 409。
- 详情读取时更新有效锚点并删除失效锚点。

前端纯函数测试：

- 从 DOM Range 计算正文字符偏移。
- 构造 32 字符前后文。
- 判断选区是否位于正文内。
- 判断选区是否与高亮重叠。
- 工具条坐标限制在视口内。

跨端验收：

- H5 Markdown 排版正确。
- H5 鼠标选择和移动端长按选择可创建、复制、取消高亮。
- 刷新和重新登录后高亮仍存在。
- 编辑正文后可匹配高亮保留，失效高亮删除。
- 微信小程序 Markdown、黄色高亮和系统复制可用。
